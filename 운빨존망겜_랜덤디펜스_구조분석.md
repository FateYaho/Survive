> 작성: 2026-04-27

## 0. 핵심 정의

운빨존많겜은 랜덤으로 유닛을 소환하고, 예측 불가능한 합성과 확률형 선택을 반복하면서, 골드·행운석·강화·배치·협동 역할 분담으로 운을 보정해 최종 웨이브까지 버티는 캐주얼 타워 디펜스 게임이다.

원랜디가 "랜덤 재료를 조합표 지식으로 해석하는 게임"이라면, 운빨존많겜은 "랜덤 결과를 계속 받아들이며 리스크를 관리하는 게임"에 가깝다. 원랜디 분석 문서에서 핵심 재미를 "운으로 받은 재료를 실력으로 해석해서 클리어 가능한 조합으로 만드는 것"으로 정리했는데, 운빨존많겜은 여기서 해석보다 확률 베팅, 즉흥 대응, 협동 보정의 비중이 더 크다.

공식 소개에서도 운빨존많겜은 유닛 소환이 운이고, 유닛 합성도 가능하지만 무엇이 나올지 모른다고 설명한다. 즉, 이 게임의 핵심은 "원하는 조합을 정확히 만드는 재미"가 아니라, 원하지 않은 결과까지 받아들여 판을 살리는 재미다.

## 1. 운빨존많겜의 핵심 재미

운빨존많겜의 재미는 다음 흐름에서 나온다.

```
랜덤 소환
→ 현재 필드 확인
→ 합성할지 유지할지 판단
→ 골드/행운석 사용 판단
→ 상위 유닛 또는 핵심 유닛 확보
→ 배치와 역할군 보완
→ 웨이브 압박 대응
→ 보스/후반 라운드 검증
→ 클리어 또는 폭사
```

핵심 감정은 "이번 판은 망한 것 같은데, 한 번만 더 뽑으면 살아날 수 있을 것 같다"이다.

| 성격 | 설명 |
|------|------|
| 운빨 게임 | 소환, 합성, 강화, 룰렛성 선택이 판의 시작점을 만든다 |
| 타워 디펜스 | 몬스터가 쌓이면 패배하므로 라인 관리가 중요하다 |
| 확률 운영 게임 | 언제 돈을 모으고, 언제 써야 하는지 판단해야 한다 |
| 리스크 관리 게임 | 합성/강화/도박을 할수록 대박과 폭망이 같이 열린다 |
| 협동 게임 | 듀오 기준으로 딜러와 서브 역할을 나누면 클리어율이 크게 오른다 |
| 성장형 모바일 게임 | 영웅, 유물, 패스, 패키지, 이벤트가 반복 플레이 동기를 만든다 |

한 줄로 줄이면:
**운빨존많겜의 재미는 운으로 망가진 판을 확률 운영과 협동으로 끝까지 살리는 데 있다.**

## 2. 전체 게임 흐름

운빨존많겜의 한 판은 크게 네 구간으로 나눌 수 있다.

| 구간 | 역할 | 플레이 성격 |
|------|------|------------|
| 초반 | 첫 안정화 | 최소 전투 유닛 확보, 초반 라인 방어 |
| 중반 | 핵심 유닛 확보 | 합성/소환/행운석으로 딜러·서포터 구성 |
| 후반 | 역할군 검증 | 딜, 스턴, 둔화, 방깎, 마나 보조 부족 여부가 드러남 |
| 최종 | 클리어 확인 | 보스와 대량 몬스터를 동시에 처리해야 함 |

커뮤니티 공략에서 보통/어려움 기준 80스테이지 클리어가 주요 목표로 다뤄지고, 어려움 공략에서도 50~60스테이지를 넘기고 80스테이지까지 가는 전략이 핵심으로 언급된다. 이는 원랜디의 "후반 검증 구간"과 비슷하게, 일정 웨이브 이후부터 조합의 빈틈이 바로 패배로 연결된다는 뜻이다.

구조적으로 보면:

```
1단계: 버틸 최소 유닛 확보
2단계: 골드/확률 강화/행운석 관리
3단계: 핵심 딜러 또는 서포터 확보
4단계: CC, 방깎, 마나 보조, 버프 보완
5단계: 후반 웨이브와 보스 압박 통과
```

## 3. 원랜디와 가장 큰 차이

| 구분 | 원랜디 | 운빨존많겜 |
|------|--------|-----------|
| 조합 방식 | 정해진 조합식 기반 | 합성 결과도 랜덤성이 강함 |
| 숙련 포인트 | 조합표 지식, 빌드 루트 이해 | 확률 판단, 돈 관리, 위기 타이밍 판단 |
| 플레이 감정 | "이 재료면 저 상위 유닛 가능" | "이번 합성에서 뭐가 나올지 모름" |
| 실패 원인 | 역할군 미완성, 조합 방향 실패 | 소환 실패, 합성 실패, 돈 사용 타이밍 실패 |
| 운 보정 | 목재, 도박, 보상 | 골드, 행운석, 확률 강화, 유물, 듀오 |
| 장기 성장 | 조합 지식과 메타 숙련 | 영웅/유물/패스/패키지 기반 성장 |

원랜디가 조합표 기반 퍼즐이라면, 운빨존많겜은 확률형 생존 운영이다. 그래서 운빨존많겜을 참고할 때 조합식만 가져오면 안 되고, 반드시 "불확실한 결과를 계속 던져주되, 플레이어가 어느 정도 복구할 수 있는 장치"까지 같이 가져와야 한다.

## 4. 핵심 루프

운빨존많겜의 코어 루프는 매우 짧다.

```
소환한다
→ 배치한다
→ 몬스터를 잡는다
→ 돈/행운석을 얻는다
→ 더 뽑거나 강화한다
→ 합성한다
→ 더 강한 유닛 또는 예상 밖 결과를 얻는다
→ 다시 버틴다
```

이 루프의 강점은 결정 주기가 짧다는 것이다. 플레이어는 계속 선택한다.

| 선택 | 질문 |
|------|------|
| 소환 | 지금 돈을 써서 유닛을 늘릴까? |
| 저축 | 위험하지 않으니 돈을 모을까? |
| 강화 | 소환 확률이나 유닛 공격력을 올릴까? |
| 합성 | 지금 있는 유닛을 합쳐서 상위 결과를 노릴까? |
| 배치 | 딜러를 앞에 둘까, CC를 길목에 둘까? |
| 협동 | 내가 딜러를 맡을까, 서브를 맡을까? |

이 구조가 모바일에 잘 맞는 이유는, 한 번의 선택이 짧고 결과가 바로 보이기 때문이다.

## 5. 전투 구조

운빨존많겜은 겉으로는 단순한 타워 디펜스지만, 실제로는 다음 역할군을 맞추는 게임이다.

| 역할 | 기능 | 부족하면 생기는 문제 |
|------|------|---------------------|
| 메인 딜러 | 몬스터 처리의 중심 | 라인이 밀림 |
| 광역 딜 | 뭉친 적 처리 | 몹이 쌓임 |
| 보스 딜 | 체력 높은 적 처리 | 보스에서 막힘 |
| 스턴/CC | 적 이동과 공격 흐름 제어 | 딜 시간이 부족함 |
| 둔화 | 몬스터 이동속도 감소 | 후반 라인 압박 증가 |
| 방어력 감소 | 물리 딜 효율 증폭 | 딜러가 있어도 딜이 안 박힘 |
| 마나 보조 | 스킬 회전율 증가 | 핵심 스킬 발동 빈도 부족 |
| 골드 보조 | 후반 성장 자금 확보 | 강화와 소환이 막힘 |

커뮤니티 공략에서도 배트맨 같은 핵심 딜러, 냥법사 같은 마나 보조, 오크주술사/호랑이사부처럼 스턴·방어력 감소·공격속도 보조 역할이 따로 언급된다. 특히 어려움 난이도에서는 스턴과 방어력 감소의 중요성이 강조된다.

**핵심: 딜러 하나만 잘 뽑는 게임이 아니라, 딜러가 일할 수 있는 환경을 확률적으로 완성하는 게임이다.**

## 6. 합성 구조

운빨존많겜의 합성은 원랜디식 조합과 다르게 작동한다.

원랜디식 조합:
```
A + B + C = 원하는 상위 유닛
```

운빨존많겜식 합성:
```
하위 유닛 압축
→ 상위 결과 기대
→ 원하는 결과가 아닐 수 있음
→ 나온 결과를 기준으로 다시 운영
```

공식 소개에서 "유닛 합성도 가능하지만 뭐가 나올지 모른다"는 식으로 안내되는 만큼, 합성은 단순 성장 수단이 아니라 도박 버튼이다.

이 구조가 좋은 이유:

| 요소 | 효과 |
|------|------|
| 결과 불확실성 | 매 판 같은 루트 반복을 막음 |
| 즉흥 대응 | 나온 유닛을 기준으로 새 전략을 세우게 함 |
| 감정 폭 | 대박/폭망 감정이 강함 |
| 판마다 서사 생성 | "망한 판을 살렸다"는 기억이 생김 |
| 반복 플레이 유도 | 다음 판에는 더 잘 뜰 것 같은 기대를 만듦 |

단점:

| 문제 | 설명 |
|------|------|
| 억울함 | 너무 랜덤이면 플레이어가 통제감을 잃음 |
| 학습 난이도 | 어떤 결과가 좋은지 모르면 판단이 어려움 |
| 밸런스 피로 | 특정 유닛이 안 뜨면 바로 망하는 구조가 될 수 있음 |
| 과금 의존 위험 | 운 보정이 유료 성장에 과하게 묶이면 피로감 증가 |

따라서 이 게임의 핵심 밸런스는 **"운이 크지만, 실력으로 조금씩 생존 확률을 올릴 수 있게 하는 것"**이다.

## 7. 자원 구조: 골드, 행운석, 강화, 유물

운빨존많겜의 자원은 단순 재화가 아니라 운 보정 수단이다.

| 자원/시스템 | 역할 |
|------------|------|
| 골드 | 유닛 소환, 강화, 위기 대응 |
| 행운석 | 고급 소환/전설급 기대값/후반 성장 |
| 확률 강화 | 소환 결과의 기대값 상승 |
| 유물 | 장기 성장, 골드/전투/확률 효율 증가 |
| 영웅 강화 | 특정 유닛의 성능 한계 상승 |
| 패스/패키지 | 성장 속도와 보상량 증가 |

공략에서는 "돈으로 때리기"가 핵심이라는 표현이 나올 정도로, 골드 운용이 중요하게 다뤄진다. 딜러는 위험하지 않으면 돈을 모으고, 서브는 안정적이지 않으면 돈을 써서 라인을 지키는 식으로 역할별 자원 사용 방식도 갈린다.

행운석도 초반에 바로 쓰기보다 모았다가 특정 타이밍에 사용하는 방식이 언급된다. 예를 들어 어려움 공략에서는 초반 행운석을 아끼고, 일정량을 모은 뒤 전설 뽑기를 노리는 운영이 제시된다.

정리하면:
```
골드 = 즉시 생존력
행운석 = 고급 결과 기대값
강화 = 장기 기대값 상승
유물 = 계정 단위 보정
과금 상품 = 성장 속도 보정
```

## 8. 웨이브 압박 구조

운빨존많겜은 몬스터 수가 쌓이면서 압박이 생긴다. 중요한 것은 "죽기 직전까지 버티다가 자원을 쓰는 운영"이 가능하다는 점이다.

공략에서는 위험 기준을 몬스터 수로 잡고, 너무 늦게 대응하면 이미 죽어 있기 때문에 죽기 전 타이밍에 소환·확률 강화·조합·합성을 빠르게 해야 한다고 설명한다. 또한 한 스테이지당 많은 몬스터가 나오기 때문에, 위기 판단이 늦으면 복구가 어렵다.

이 구조의 재미:

| 압박 요소 | 플레이 감정 |
|-----------|------------|
| 몬스터가 쌓임 | "아직 버틸 수 있나?" |
| 돈을 모음 | "조금만 더 참으면 큰 선택 가능" |
| 갑자기 밀림 | "지금 써야 한다" |
| 합성 대박 | "살았다" |
| 합성 실패 | "판 터졌다" |

**즉, 운빨존많겜은 안전할 때 돈을 모으고, 위험할 때 확률을 터뜨리는 게임이다.**

## 9. 협동 구조

운빨존많겜의 중요한 차별점은 듀오 플레이에서 나온다.

커뮤니티 공략에서는 듀오가 없어도 가능하지만 있으면 클리어 확률이 크게 올라간다고 설명하고, 기본 역할을 딜러와 서브로 나눈다. 더 좋은 유물과 강화 상태를 가진 쪽이 딜러를 맡고, 부족한 쪽이 서브를 맡는 식이다.

| 역할 | 핵심 임무 |
|------|-----------|
| 딜러 | 돈을 모으고 핵심 딜러를 키워 후반 화력을 책임짐 |
| 서브 | 초중반 안정화, 소환, CC/보조 유닛 확보 |
| 공통 | 위험할 때 합성/소환/강화로 라인 복구 |

이 구조가 좋은 이유:
1. 실력이 부족한 유저도 서브 역할로 참여할 수 있다.
2. 고성장 유저는 딜러 역할로 캐리 감정을 얻는다.
3. 같은 판에서도 역할이 달라져 반복 피로가 줄어든다.
4. 친구 초대, 커뮤니티 모집, 재방문 동기가 생긴다.

혼자 개발할 때 이 구조를 그대로 넣기는 부담이 크다. 대신 다음처럼 축소할 수 있다.

| 원본 구조 | 축소 적용안 |
|-----------|------------|
| 실시간 듀오 | AI 파트너 |
| 딜러/서브 분담 | 메인 보드/서포트 보드 |
| 파트너 성장 차이 | 선택 가능한 보조 특성 |
| 협동 보상 | 싱글 클리어 보조 보상 |

## 10. 영웅/유닛 구조

운빨존많겜의 유닛 구조는 단순 등급 상승보다 역할 기반 가치가 중요하다.

| 유닛 가치 | 설명 |
|-----------|------|
| 초반 안정 유닛 | 초반 라인을 막는 데 중요 |
| 핵심 딜러 | 후반 클리어의 중심 |
| 마나 보조 | 스킬 기반 딜러의 효율 증가 |
| 스턴/둔화 | 몬스터를 붙잡아 딜 시간을 늘림 |
| 방깎/버프 | 딜러의 실질 화력을 증폭 |
| 골드/경제 유닛 | 후반 자원 차이를 만듦 |

배트맨은 강화 리스크가 있지만 강한 딜러로 소개되고, 냥법사는 아군 마나 회복 보조로, 오크주술사와 호랑이사부는 스턴·방어력 감소·공격속도 증가 같은 보조 역할로 언급된다. 특히 후반에는 방어력 감소 영웅을 많이 모으는 것이 중요하다는 공략도 있다.

**중요한 점: 운빨존많겜의 좋은 유닛은 "혼자 강한 유닛"이 아니라, 판 전체의 확률과 전투 효율을 올려주는 유닛이다.**

---

## 11. 초보가 막히는 이유

초보가 막히는 이유는 단순히 "운이 나빠서"가 아니다.

| 막히는 지점 | 초보가 모르는 것 | 결과 |
|------------|----------------|------|
| 돈 사용 | 언제 모으고 언제 써야 하는지 모름 | 초반에 돈을 다 써서 후반 약함 |
| 합성 판단 | 지금 합쳐야 하는지 유지해야 하는지 모름 | 좋은 역할군을 날림 |
| 배치 | 딜러/CC/서포터 위치를 모름 | 딜 효율이 낮음 |
| 역할군 | 딜러만 있으면 된다고 생각함 | 후반에 라인이 밀림 |
| 행운석 | 초반에 바로 써버림 | 고급 선택 타이밍을 놓침 |
| 협동 | 딜러/서브 역할을 모름 | 둘 다 돈을 쓰거나 둘 다 모으다 터짐 |
| 장기 성장 | 유물/영웅 강화 중요성을 모름 | 같은 운영을 해도 화력이 부족함 |

초보는 보통 이렇게 생각한다: `많이 뽑으면 강해지겠지?`

하지만 실제 공식은 다르다:
```
적절한 타이밍까지 버티기
+ 돈과 행운석 아끼기
+ 핵심 딜러 확보
+ CC/방깎/마나 보조 완성
+ 위기 타이밍에 자원 폭발
= 클리어 확률 상승
```

## 12. BM/라이브 구조

운빨존많겜은 모바일 라이브 게임 구조가 강하다. App Store에는 랜덤 박스, 광고, 메시지 및 채팅, 앱 내 구입이 표시되며, 인앱 상품으로 Wave Pass, VVIP Package, Starter Package, Upgrade Trigger Package, Treasure Open Pack 등이 노출된다.

BM 구조를 게임 루프와 연결하면:
```
한 판 플레이
→ 보상 획득
→ 영웅/유물/계정 성장
→ 더 높은 난이도 도전
→ 막힘
→ 패스/패키지/광고/이벤트 보상으로 성장 보정
→ 재도전
```

| BM 요소 | 설계상 역할 |
|---------|------------|
| 광고 | 무료 유저 보상 보정 |
| 패스 | 장기 접속/반복 플레이 유도 |
| 스타터 패키지 | 초반 진입 장벽 완화 |
| 업그레이드 패키지 | 막힌 구간 돌파 욕구 대응 |
| 랜덤 박스 | 수집/성장 기대감 제공 |
| 이벤트 | 복귀 명분과 신규 목표 제공 |

**운빨존많겜의 BM은 단순히 돈을 받는 구조가 아니라, 운으로 막힌 감정을 성장 보정 상품으로 연결하는 구조다.**

## 13. 온보딩 구조

운빨존많겜의 초반 진입은 직관적이다.

```
소환한다 → 배치한다 → 몬스터를 잡는다 → 합성한다 → 더 강해진다
```

이 단계는 누구나 이해할 수 있다. 하지만 중반 이후부터는 숨은 지식이 필요해진다.

| 초반에 보이는 것 | 실제로 중요한 것 |
|----------------|----------------|
| 유닛 많이 뽑기 | 언제 뽑을지 |
| 합성하기 | 무엇을 남기고 무엇을 합칠지 |
| 강한 유닛 얻기 | 역할군을 맞출지 |
| 몬스터 잡기 | 돈을 모으면서 얼마나 버틸지 |
| 듀오 플레이 | 누가 딜러고 누가 서브인지 |

이 게임의 온보딩 강점은 처음 1분 안에 재미가 보인다는 점이다. 반대로 약점은 왜 졌는지 명확하게 알려주지 않으면 초보가 "운 때문에 졌다"고만 느낄 수 있다는 점이다.

혼자 개발할 때 반드시 넣어야 할 UX:

| UX 장치 | 이유 |
|---------|------|
| 현재 부족한 역할 표시 | "딜 부족/CC 부족/보스딜 부족"을 알려줘야 함 |
| 합성 예상 범위 표시 | 완전 랜덤이어도 기대 등급은 보여줘야 함 |
| 위험도 게이지 | 몬스터 수가 언제 위험한지 알려줘야 함 |
| 자원 추천 | "지금은 저장/지금은 소환" 힌트 |
| 패배 원인 리포트 | 다음 판 개선 포인트 제공 |

## 14. 실패 구조

운빨존많겜의 실패는 크게 다섯 가지다.

| 실패 유형 | 원인 | 플레이어 감정 |
|-----------|------|--------------|
| 초반 라인사 | 초반 유닛/배치 실패 | "시작부터 안 풀렸다" |
| 중반 화력 부족 | 핵심 딜러 부재 | "딜러가 안 떴다" |
| 후반 제어 실패 | 스턴/둔화/마나 보조 부족 | "딜할 시간이 없다" |
| 보스 실패 | 단일/폭딜 부족 | "잡몹은 되는데 보스가 안 죽는다" |
| 경제 실패 | 돈/행운석을 너무 빨리 씀 | "후반에 할 게 없다" |

좋은 실패는 플레이어가 이렇게 생각하게 만든다:
```
이번 판은 내가 돈을 너무 빨리 썼다.
이번 판은 CC를 너무 늦게 챙겼다.
이번 판은 딜러만 보고 서포터를 버렸다.
```

나쁜 실패는 이렇게 느끼게 만든다:
```
그냥 운이 없어서 졌다.
뭘 해도 안 된다.
```

**운빨존많겜류 게임을 만들 때 가장 중요한 기준: 운으로 졌다고 느끼게 하되, 다음 판에는 내가 더 잘할 수 있다고 믿게 만들어야 한다.**

## 16. MVP 축소 설계안

혼자 개발한다면 운빨존많겜을 그대로 만들지 말고 다음 정도로 줄이는 게 좋다.

| 항목 | 추천 규모 |
|------|-----------|
| 기본 유닛 | 6~8개 |
| 중급 유닛 | 6~8개 |
| 고급 유닛 | 5~6개 |
| 전설/신화급 | 3~4개 |
| 최종 유닛 | 1~2개 |
| 총 유닛 수 | 20~28개 |
| 웨이브 | 30~40웨이브 |
| 플레이타임 | 10~15분 |
| 핵심 자원 | 골드 + 행운석 2종 |
| 역할군 | 딜러/광역/보스딜/CC/버프/경제 |
| 모드 | 싱글 우선, 협동은 후순위 |

추천 MVP 루프:
```
골드로 랜덤 소환
→ 같은 등급 3개 합성
→ 결과는 같은 등급 상위풀 또는 다음 등급 랜덤
→ 행운석으로 고급 소환/확률 강화
→ 10웨이브마다 보스
→ 30~40웨이브 클리어
```

## 17. 네 게임에 적용할 수 있는 핵심 공식

운빨존많겜식 재미를 가져오려면 다음 네 가지가 필요하다.

```
랜덤 결과
+ 짧은 선택 주기
+ 위기 전까지 버티는 자원 운영
+ 역할군 완성
= 운빨 디펜스의 재미
```

더 기획적으로 쓰면:

| 설계 축 | 질문 |
|---------|------|
| 랜덤 | 매 판 다른 상황이 만들어지는가? |
| 통제 | 운이 나빠도 복구할 선택지가 있는가? |
| 압박 | 돈을 모으다 터질 수 있는 긴장감이 있는가? |
| 역할군 | 좋은 유닛 하나만으로 클리어되지 않는가? |
| 성장 | 다음 판에 더 잘할 이유가 있는가? |
| UX | 왜 졌는지 알 수 있는가? |

## 18. 최종 정리

운빨존많겜의 핵심은 "운빨"이라는 단어에 있지만, 실제 재미는 단순 운이 아니다.

진짜 핵심은 다음이다:
```
랜덤으로 유닛을 받고,
합성 결과도 예측할 수 없지만,
돈과 행운석을 아끼고 쓰는 타이밍,
딜러와 서포터 역할 분담,
CC/방깎/마나 보조 같은 역할군 완성,
위기 직전의 자원 폭발로
망한 판을 살리는 게임.
```

원랜디가 조합표를 아는 재미라면, 운빨존많겜은 판이 터지기 직전까지 확률을 굴리는 재미다.

**한 줄 결론:**
운빨존많겜식 재미를 만들려면 랜덤 소환만 넣으면 안 되고, "언제 참을지, 언제 터뜨릴지, 실패한 운을 어떻게 복구할지"까지 설계해야 한다.
