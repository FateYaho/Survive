# 원랜디식 랜덤 조합 디펜스 구조 분석 문서

## 0. 문서 목적

이 문서는 원피스 랜덤 디펜스(원랜디)류 게임을 참고해 혼자 새로운 랜덤 조합 디펜스 게임을 만들 때, 막히는 지점마다 다시 확인할 수 있는 기준 문서다.

목적은 원랜디를 그대로 복제하는 것이 아니라, 원랜디가 왜 재미있는지, 어떤 구조로 클리어가 만들어지는지, 혼자 개발할 때 어떤 요소를 가져가고 어떤 요소를 줄여야 하는지를 정리하는 데 있다.

핵심 정의는 다음과 같다.

> 원랜디는 랜덤으로 얻은 유닛을 조합해 상위 유닛을 만들고, 물딜/마딜 중 하나의 클리어 방향에 맞춰 스턴, 이감, 라인딜, 보잡, 방깎 또는 체력비례딜을 완성해 최종 라운드까지 라인과 보스를 버티는 조합 운영 디펜스 게임이다.

---

## 1. 원랜디의 핵심 재미

원랜디의 재미는 단순히 강한 유닛을 만드는 데 있지 않다. 진짜 재미는 다음 흐름에서 나온다.

```
랜덤 유닛 획득
→ 현재 패 확인
→ 가능한 조합 판단
→ 빌드 방향 결정
→ 상위 유닛 완성
→ 부족한 역할 보완
→ 라운드별 컷 통과
→ 최종 클리어
```

즉, 원랜디는 단순 타워디펜스가 아니라 다음 성격이 섞여 있다.

| 성격 | 설명 |
|------|------|
| 랜덤 게임 | 매 판 시작 패와 보상이 달라짐 |
| 조합 퍼즐 | 가진 유닛을 어떤 상위 유닛으로 압축할지 판단 |
| 운영 게임 | 물딜/마딜, 스턴, 이감, 보잡 등 필요한 역할을 맞춤 |
| 지식 게임 | 조합식, 상위 루트, 클리어 컷을 아는 사람이 유리 |
| 디펜스 게임 | 라인몹과 보스를 막지 못하면 실패 |

핵심 재미는 "운으로 받은 재료를 실력으로 해석해서 클리어 가능한 조합으로 만드는 것"이다.

---

## 2. 전체 게임 흐름

원랜디의 한 판은 크게 두 구간으로 나뉜다.

| 구간 | 역할 | 플레이 성격 |
|------|------|------------|
| 1~50라운드 | 빌드업 구간 | 조합, 스토리, 미션, 보상 획득 |
| 50라운드 이후 | 검증 구간 | 스턴, 이감, 보잡, 딜 부족 여부가 드러남 |

현재 기준으로는 65라운드가 끝이라고 보면 된다. 실질적인 구조는 다음과 같다.

```
1~50라운드: 패를 모으고 조합 방향을 잡는 구간
50~65라운드: 만든 조합이 클리어 가능한지 검증받는 구간
```

50라운드 전까지는 다소 어설픈 조합이어도 진행할 수 있지만, 50라운드 이후부터는 부족한 역할이 바로 실패 원인이 된다.

## 3. 등급별 조합 구조

원랜디 조합 구조는 단순히 낮은 등급에서 높은 등급으로 올라가는 구조가 아니다. 각 등급마다 플레이에서 맡는 역할이 다르다.

| 등급 | 대략 개수 | 기본 구조 | 역할 |
|------|----------|----------|------|
| 안흔함 | 14 | 흔함 2개 | 초반 중복 정리 |
| 특별함 | 33 | 흔함 3개 또는 안흔함+흔함 | 첫 빌드 방향 생성 |
| 희귀함 | 44 | 특별함 2~3개 중심 | 중반 목표 유닛 |
| 전설적인 | 41 | 희귀함 3개 + 목재 | 후반 핵심 재료 |
| 히든 | 22 | 희귀/전설/특수 유닛 조합 | 지식 보상, 특수 루트 |
| 초월 | 29 | 전설급 재료 + 공통 게이트 재료 | 메인 캐리 진입 |
| 불멸 | 12 | 상위 재료 3개 + 목재 | 최상위 전투력 |
| 영원 | 12 | 상위 재료 3~7개 + 목재/특수 아이템 | 최종 목표 |
| 제한 | 10 | 특정 시작 유닛 + 비용/조건 | 조건형 특수 루트 |

전체 흐름은 다음과 같다.

```
흔함 랜덤 획득
↓
안흔함: 중복 정리
↓
특별함: 빌드 방향 생성
↓
희귀함: 중반 목표
↓
전설적인: 후반 핵심 재료
↓
히든/초월/제한: 특수 분기
↓
불멸/영원: 최종 목표
```

---

## 4. 등급별 역할 해석

### 4.1 안흔함: 초반 정리 단계

안흔함은 대부분 흔함 2개로 만들어진다.

예시 구조:
```
같은 흔함 2개 → 안흔함
서로 다른 흔함 2개 → 안흔함
기본 병사 2개 → 안흔함
```

안흔함의 목적은 강력한 유닛을 만드는 것이 아니라, 초반 랜덤 유닛을 정리하고 조합 시스템을 학습시키는 것이다.

플레이어 감정: "아, 이 게임은 나온 유닛을 조합해서 키우는 게임이구나."

혼자 개발할 때 가져갈 요소: 2개 조합, 같은 유닛 2개 합성, 직관적인 결과, 복잡한 조건 없음

### 4.2 특별함: 첫 빌드 방향 단계

특별함부터 유저가 "어떤 방향으로 갈지" 판단하기 시작한다.

대표 구조:
```
흔함 3개 → 특별함
안흔함 2개 + 흔함 → 특별함
관계 있는 유닛 조합 → 특별함
```

특별함은 초반 패를 특정 방향으로 묶어주는 단계다.

플레이어 감정: "이거 모으면 저 캐릭으로 갈 수 있네?"

혼자 개발할 때 가져갈 요소: 동일 유닛 3개 승급, 2~3개 재료 조합, 하위 재료 3~5개 수준의 부담, 빌드 방향이 보이는 조합

### 4.3 희귀함: 중반 목표 단계

희귀함부터는 조합이 본격적인 목표가 된다.

대표 구조:
```
특별함 + 특별함 + 특별함 → 희귀함
특별함 2개 + 보조재료 → 희귀함
```

희귀함은 보통 하위 재료 기준 10~15개 정도의 가치가 들어간다. 이 단계에서 유저는 "이번 판은 이 루트로 가야겠다"라고 판단한다.

플레이어 감정: "이번 판 방향이 잡혔다."

### 4.4 전설적인: 후반 핵심 재료 단계

전설적인 등급은 대부분 다음 구조를 가진다.

```
희귀함 3개 + 목재 3개 → 전설적인
```

전설은 단순히 강한 유닛이 아니라 상위 조합의 핵심 재료다.

플레이어 감정: "이제 상위 조합을 노릴 수 있다."

전설 단계부터 목재가 본격적으로 조합 비용으로 들어가며, 조합이 단순 유닛 압축에서 자원 운영까지 확장된다.

### 4.5 히든: 지식 보상 단계

히든은 일반 성장 루트와 다르게, 알고 있는 사람에게 보상을 주는 구조다.

대표 구조:
```
희귀/전설/특수 유닛 조합 → 히든
```

히든의 재미는 성능뿐 아니라 "내가 이 조합을 알고 있다"는 숙련감에서 나온다.

플레이어 감정: "나 이거 만들 줄 안다."

혼자 개발할 때는 히든을 많이 넣기보다 3~5개 정도만 두는 것이 좋다. 너무 많으면 조합표 의존도가 커지고 개발 부담이 커진다.

### 4.6 초월: 메인 캐리 진입 단계

초월은 상위 캐리 유닛으로 들어가는 관문이다. 핵심은 공통 게이트 재료가 있다는 점이다.

```
전설급 재료 여러 개 + 공통 게이트 재료 → 초월
```

원랜디에서는 초월 쿠마가 이 역할을 한다.

이 구조가 좋은 이유:

| 요소 | 효과 |
|------|------|
| 공통 재료 | 상위 진입 속도 제어 |
| 목표 집중 | 유저가 특정 핵심 재료를 노리게 됨 |
| 밸런스 제어 | 초월 남발 방지 |
| 성취감 | 완성 시 큰 파워 상승 |

네 게임에서는 "각성 코어", "고대 문장", "영웅의 증표" 같은 공통 게이트 재료로 바꿀 수 있다.

### 4.7 불멸: 최상위 전투력 단계

불멸은 구조가 비교적 단순하다.

```
상위 재료 3개 + 목재 10개 → 불멸
```

불멸은 복잡한 조합보다 무거운 비용과 높은 전투력이 핵심이다.

플레이어 감정: "이거 만들면 이제 버틴다."

혼자 개발할 때는 불멸과 영원을 따로 둘 필요 없이 "최종 등급" 하나로 합쳐도 된다.

### 4.8 영원: 최종 목표 단계

영원은 심리적으로 최종 목표에 해당한다.

```
상위 재료 3~7개 + 목재 5개 + 특수 아이템 → 영원
```

영원 조합은 재료가 길고, 특정 아이템이나 특수 조건이 붙기도 한다.

플레이어 감정: "이번 판 목표는 영원 하나 완성이다."

영원은 반드시 많이 필요하지 않다. 오히려 1~2개만 있어도 강한 목표감을 줄 수 있다.

### 4.9 제한: 조건형 특수 루트

제한은 단순 조합이 아니라 조건형 조합이다.

| 조건 유형 | 예시 |
|----------|------|
| 골드 비용 | 5,000골드, 30,000골드 |
| 목재 비용 | 목재 5개 |
| 라운드 제한 | 특정 라운드 이전 가능 |
| 아이템 필요 | 불사조의 깃털 등 |
| 특성 포인트 | 특성포인트 요구 |

제한은 플레이 중간에 특수 목표를 만드는 역할을 한다. 혼자 개발할 때는 1~2개만 넣어도 충분하다.

## 5. 클리어 필수 역할군

원랜디는 좋은 유닛 하나만 만든다고 깨지는 게임이 아니다. 실제 클리어는 필수 역할군을 맞추는 게임이다.

| 역할 | 기능 | 부족하면 생기는 문제 |
|------|------|-------------------|
| 메인딜 | 라인몹을 처리하는 주력 딜 | 잡몹이 쌓여 라인사 |
| 스턴 | 몹을 멈춰 딜 시간을 확보 | 라인이 빠르게 밀림 |
| 이감 | 몹 이동속도를 낮춤 | 스턴 공백에 라인 밀림 |
| 보잡 | 보스 처리 | 보스 라운드에서 사망 |
| 방깎 | 물리딜 효율 증폭 | 물딜이 제대로 박히지 않음 |
| 단일 | 피 많은 적을 체력비례로 깎음 | 고체력 몹이 남음 |
| 끝딜 | 피 적은 적을 체력비례로 마무리 | 딸피 몹이 누적됨 |

실패는 크게 두 가지다.

| 실패 유형 | 원인 | 부족한 역할 |
|----------|------|-----------|
| 라인사 | 잡몹이 밀려서 죽음 | 메인딜, 스턴, 이감, 방깎, 단일/끝딜 |
| 보스사 | 보스를 못 잡아서 죽음 | 보잡, 단일 집중딜 |

## 6. 물딜 클리어 구조

물딜은 방어력 감소, 즉 방깎을 바탕으로 물리딜을 증폭시키는 구조다.

| 역할 | 필요량 | 설명 |
|------|--------|------|
| 스턴 | 약 1.5스턴 | 라인 홀딩 |
| 보잡 | 1캐릭 | 보스 처리 |
| 메인딜 | 1~2개 | 주력 라인딜 |
| 방깎 | 다수 | 물리딜 효율 상승 |
| 이감 | 필요 | 라인 유지 보조 |

물딜의 핵심: 메인딜은 1~2개면 되고, 나머지는 방깎/이감/스턴/보잡으로 딜러가 제대로 때릴 환경을 만드는 구조다. 물딜은 강한 딜러를 많이 만드는 게임이 아니라, 한두 개의 메인딜이 최대 효율을 내도록 주변 유틸을 맞추는 게임이다.

---

## 7. 마딜 클리어 구조

마딜은 마법딜과 체력비례 딜 역할을 나눠서 몹을 처리하는 구조다.

| 역할 | 필요량 | 설명 |
|------|--------|------|
| 스턴 | 약 2스턴 | 더 긴 홀딩 필요 |
| 보잡 | 1캐릭 | 보스 처리 |
| 메인딜 | 1~2개 | 주력 마법딜 |
| 단일 | 필요 | 남은 체력 기준 % 데미지, 피 많은 적 처리 |
| 끝딜 | 필요 | 전체 체력 기준 % 데미지, 피 적은 적 마무리 |
| 이감 | 필요 | 딜 시간 확보 |

마딜의 핵심: 메인딜만으로 미는 것이 아니라, 단일과 끝딜이 체력 구간별로 역할을 나눠 몹을 처리하는 구조다. 마딜은 물딜보다 역할이 더 세분화되어 있고, 어떤 유닛이 어떤 체력 구간을 담당하는지 이해해야 한다.

---

## 8. 물딜과 마딜의 차이

| 구분 | 물딜 | 마딜 |
|------|------|------|
| 핵심 방식 | 방깎 후 물리딜 | 마법딜 + 체력비례딜 |
| 필수 유틸 | 방깎, 스턴, 이감 | 스턴, 이감, 단일, 끝딜 |
| 보스 처리 | 보잡 유닛 중요 | 보잡 유닛 중요 |
| 실패 원인 | 방깎/보잡/스턴 부족 | 단일/끝딜/스턴 부족 |
| 운영 느낌 | 효율 세팅형 | 역할 분업형 |
| 초보 난이도 | 방깎 개념 이해 필요 | 단일/끝딜 개념 이해 필요 |

간단히 정리하면:
```
물딜 = 방깎으로 딜을 증폭시키는 조합
마딜 = 체력 구간별 딜 역할을 나눠 처리하는 조합
```

## 9. 라운드별 운영 구조

현재 기준으로 65라운드 종료 구조를 기준으로 정리한다.

| 라운드 | 목표 | 설명 |
|--------|------|------|
| 1~50라 | 조합, 스토리, 미션 | 패를 모으고 방향을 잡는 구간 |
| 50라 이후 | 스턴/이감/보잡 완성 | 실제 클리어 검증 시작 |
| 50~65라 | 5라운드마다 보스 대응 | 라인과 보스를 동시에 봐야 함 |
| 65라 | 최종 클리어 | 딜/유틸/보잡이 모두 맞으면 클리어 |

라운드 구조의 핵심: 50라운드 전까지는 준비 구간이고, 50라운드 이후부터는 조합의 빈틈을 검사하는 구간이다. 50라운드 이후에 딜이 부족하면 라인이 밀려 죽고, 보잡이 부족하면 보스에서 죽는다.

---

## 10. 자원: 목재, 위습, 도박, 보상

목재는 단순 조합 재료가 아니라 운을 보정하는 핵심 자원이다.

| 요소 | 설명 | 역할 |
|------|------|------|
| 목재 | 미션 클리어, 보상 등으로 획득 | 조합, 도박, 보정에 사용 |
| 위습 | 라운드 끝에 나오는 2마리 | 도박 기회 제공 |
| 하급 도박 (하도) | 흔함/안흔함 패 보충 |  |
| 중급 도박 (중도) | 중간 재료 보충 |  |
| 고급 도박 (고도) | 희귀 이상 노림 |  |
| 보상 | 스토리/미션 진행으로 획득 | 빌드 방향 결정에 영향 |

이 시스템의 역할:
```
랜덤으로 패가 나옴
→ 조합 방향 판단
→ 부족한 재료를 도박/보상으로 보정
→ 클리어 역할군 완성
```

랜덤이 너무 강하면 억울해지기 때문에, 목재와 도박은 꼬인 판을 복구할 수 있는 완충 장치 역할을 한다.

---

## 11. 초보가 막히는 이유

초보가 막히는 이유는 단순히 조합식을 몰라서가 아니다. 더 정확히는 "어떤 역할을 완성해야 클리어되는지"를 모르기 때문이다.

| 막히는 지점 | 초보가 모르는 것 | 결과 |
|-----------|---------------|------|
| 조합 목표 | 어떤 상위를 가야 할지 모름 | 패를 아무렇게나 사용 |
| 역할군 | 스턴/이감/보잡/방깎 필요성을 모름 | 딜러만 만들고 죽음 |
| 라운드 컷 | 50라 이후 필요한 최소 세팅을 모름 | 갑자기 라인사/보스사 |
| 자원 운영 | 목재를 어디에 써야 할지 모름 | 필요한 패 보정 실패 |
| 빌드 선택 | 물딜/마딜 방향을 못 정함 | 중간에 조합이 꼬임 |

초보는 보통 이렇게 생각한다: "강한 캐릭 만들면 깨겠지?"

하지만 실제 클리어 공식은 다르다:
> **강한 캐릭 + 필수 역할군 + 라운드별 컷 통과 = 클리어**

이 차이가 초보 진입장벽의 핵심이다.

---

## 12. 원랜디식 재미의 핵심 정리

### 12.1 랜덤이 시작점을 만든다
매 판 시작 패가 다르기 때문에 같은 조합만 반복하지 않는다.
```
랜덤 뽑기 → 나온 재료 확인 → 가능한 조합 판단
```

### 12.2 조합표가 목표를 만든다
조합표는 단순 정보가 아니라 목표 생성 장치다.
```
이 재료를 모으면 특별함
특별함을 모으면 희귀함
희귀함을 모으면 전설
전설을 모으면 초월/불멸/영원
```

### 12.3 등급 상승이 성취감을 만든다
많은 하위 재료가 하나의 상위 유닛으로 압축된다. 이 압축감이 성취감을 만든다.

### 12.4 역할군 완성이 클리어감을 만든다
유닛 등급이 높다고 무조건 깨는 것이 아니라, 필요한 역할을 맞췄을 때 클리어가 가능해진다.

### 12.5 히든/제한이 숙련도 차이를 만든다
초보는 보이는 조합만 하고, 숙련자는 히든/제한/특수 조건까지 고려한다. 이 차이가 장기 플레이 동기를 만든다.

---

## 13. 혼자 개발할 때 가져갈 핵심

원랜디를 그대로 만들면 너무 크다. 혼자 개발할 때는 구조만 가져오고 규모는 줄여야 한다.

**반드시 가져갈 것**

| 요소 | 이유 |
|------|------|
| 랜덤 유닛 획득 | 매 판 다른 상황을 만듦 |
| 2~3개 재료 조합 | 이해하기 쉬움 |
| 등급 압축 구조 | 성장감 제공 |
| 물딜/마딜 같은 빌드 타입 | 전략 선택 제공 |
| 필수 역할군 | 클리어 조건을 깊게 만듦 |
| 보상/도박 보정 | 운빨 스트레스 완화 |
| 소수의 히든/조건 조합 | 숙련도 차이 제공 |

**줄여야 할 것**

| 요소 | 이유 |
|------|------|
| 지나치게 많은 유닛 수 | 밸런스와 개발량 증가 |
| 너무 많은 등급 | 유저가 헷갈림 |
| 외부 조합표 의존 | 진입장벽 상승 |
| 복잡한 메타 용어 | 초보 이탈 |
| 긴 플레이타임 | 혼자 개발 MVP에 부담 |
| 과도한 특수 조건 | 디버깅과 밸런스 부담 |

## 14. 혼자 개발용 축소 구조 제안

원랜디 등급을 그대로 쓰지 말고 다음처럼 압축하는 것이 좋다.

| 원랜디 구조 | 축소 버전 |
|-----------|---------|
| 흔함 | 기본 유닛 |
| 안흔함 | 1차 조합 |
| 특별함 | 2차 조합 |
| 희귀함 | 고급 유닛 |
| 전설적인 | 전설 유닛 |
| 히든/초월 | 각성 유닛 |
| 불멸/영원 | 최종 유닛 |
| 제한 | 특수 조건 유닛 |

추천 흐름:
```
기본 유닛
↓
1차 조합
↓
2차 조합
↓
고급 유닛
↓
전설 유닛
↓
각성/최종 유닛
```

## 15. MVP 규모 제안

처음 버전은 작게 만들어야 한다.

| 항목 | 추천 규모 |
|------|---------|
| 기본 유닛 | 8~12개 |
| 1차 조합 | 6~8개 |
| 2차 조합 | 10~12개 |
| 고급 유닛 | 8~12개 |
| 전설 유닛 | 5~8개 |
| 히든/각성 | 3~5개 |
| 최종 유닛 | 1~2개 |
| 특수 조건 유닛 | 1~2개 |
| 총 조합 수 | 약 40~60개 |
| 플레이타임 | 15~25분 |

핵심 검증 질문: **유저가 조합 하나를 완성하려고 다음 판을 다시 누르는가?** — 이게 되면 확장하면 된다.

## 16. 네 게임용 필수 역할 구조

원랜디 역할군을 그대로 가져오면 복잡하므로 다음처럼 압축할 수 있다.

| 원랜디 역할 | 네 게임용 역할 |
|-----------|-------------|
| 메인딜 | 라인딜 |
| 보잡 | 보스딜 |
| 스턴 | 군중제어 |
| 이감 | 둔화 |
| 방깎 | 방어 감소/저항 감소 |
| 단일 | 고체력 대상 처리 |
| 끝딜 | 마무리 딜 |

MVP 기준 필수 역할은 다음 4~5개면 충분하다:
```
라인딜 / 보스딜 / 군중제어 / 둔화 / 디버프
```

물딜/마딜을 유지한다면:

| 빌드 타입 | 핵심 구조 |
|----------|---------|
| 물리 빌드 | 방깎 + 메인딜 + 보스딜 |
| 마법 빌드 | 광역딜 + 체력비례딜 + 군중제어 |

## 17. 막힐 때 확인할 체크리스트

### 17.1 조합 시스템이 막힐 때
- 이 조합은 유저가 직관적으로 이해할 수 있는가?
- 재료 수가 너무 많지 않은가?
- 하위 재료가 상위 유닛으로 압축되는 성취감이 있는가?
- 같은 유닛 중복을 처리할 방법이 있는가?
- 조합표 없이도 어느 정도 방향을 알 수 있는가?
- 히든 조합은 너무 많지 않은가?

### 17.2 게임이 재미없게 느껴질 때
- 매 판 다른 선택이 생기는가?
- 유저가 "이번 판 목표"를 잡을 수 있는가?
- 중간에 조합 완성 성취감이 있는가?
- 라운드별 압박이 있는가?
- 실패 이유가 명확한가?
- 운이 나빠도 복구 수단이 있는가?

### 17.3 클리어 구조가 막힐 때
- 라인사와 보스사를 구분할 수 있는가?
- 라인딜과 보스딜이 다른 역할로 존재하는가?
- 스턴/이감 같은 생존 유틸이 필요한가?
- 딜러만 많이 만들면 깨지는 구조가 되어버리진 않았는가?
- 50라운드 이후처럼 검증 구간이 존재하는가?

### 17.4 초보 UX가 막힐 때
- 유저가 어떤 유닛을 가야 할지 알 수 있는가?
- 현재 조합에 부족한 역할을 알 수 있는가?
- 조합 가능 유닛을 게임 안에서 보여주는가?
- 실패했을 때 왜 졌는지 알려주는가?
- 물딜/마딜 같은 빌드 방향을 쉽게 이해시킬 수 있는가?

### 17.5 밸런스가 막힐 때
- 메인딜 1~2개만으로 충분한가?
- 보스딜이 없으면 보스에서 막히는가?
- 군중제어가 없으면 라인이 밀리는가?
- 디버프 없이 딜이 너무 잘 들어가진 않는가?
- 특정 조합 하나만 정답이 되지는 않는가?
- 운이 나쁜 판도 보정 자원으로 복구 가능한가?

## 18. 최종 정리

원랜디의 핵심은 "조합식이 많다"가 아니다. 진짜 핵심은 다음이다.

```
랜덤으로 받은 재료를
조합표 기반으로 해석하고,
상위 유닛으로 압축하면서,
클리어에 필요한 역할군을 완성하고,
라운드별 컷을 넘기는 게임.
```

**반드시 남길 것**

| 반드시 남길 것 | 이유 |
|-------------|------|
| 랜덤 재료 | 매 판 다른 상황 생성 |
| 조합 압축 | 성장감과 목표감 생성 |
| 역할군 클리어 조건 | 단순 스펙 게임 방지 |
| 운 보정 자원 | 억울한 실패 완화 |

**줄일 것**

| 줄일 것 | 이유 |
|--------|------|
| 과도한 조합 수 | 개발 부담 증가 |
| 너무 많은 등급 | 유저 혼란 |
| 외부 공략 의존 | 초보 진입장벽 |
| 복잡한 상위 조건 | 밸런스와 디버깅 부담 |

한 줄로 정리하면:

> **원랜디식 재미를 살리려면 "랜덤 조합"만 베끼면 안 되고, "역할군을 맞춰 클리어 각을 만드는 운영 구조"까지 가져와야 한다.**
