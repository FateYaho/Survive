## 🧮 2. 수입 계산 공식

### 2-A. DAY 플레이어 채집 수입

**한 타일의 총 자원량** = `initialResourcePerTile` = 8 (W 또는 S)

**한 타일에서 캐는 시간**:
- 나무 = `4 collects × 1s` = **4초**
- 돌 = `4 collects × 1.5s` = **6초**

**타일 간 이동 시간** (간격 2칸 보장):
- 평균 이동거리 = ~2 타일 = `2 / 4 (속도)` = **0.5초**

**1 타일 당 사이클**:
- 나무: 4 + 0.5 = **4.5초** → 8W
- 돌: 6 + 0.5 = **6.5초** → 8S

**90초 동안 캘 수 있는 타일 수**:
- 나무만: 90 / 4.5 ≈ 20 타일 → 160W (이론치, 현실 더 적음)
- 돌만: 90 / 6.5 ≈ 13 타일 → 104S
- 혼합 (W 65%, S 35%): 약 12 W + 4 S 타일 → **96W + 32S**

**현실 보정 계수**:
- 시야 4타일 = 9×9 = 81 타일 가시 영역. 자원 밀도 15% → ~12 타일 즉시 발견 가능.
- 한 사이클 내 시야 확장(이동) 고려해 ~15~20 타일 발견 가능.
- 플레이어 행동 비효율(이동 결정·길 헤맴) 약 20~30% 손실.
- **DAY 1 현실 추정**: 50~80W + 25~40S
- **DAY 2 이후** (영토 더 넓음): 60~100W + 30~50S

### 2-B. NIGHT 몬스터 드롭 수입

`drop_per_wolf × monsters_in_cycle × kill_rate`

| Cycle | 100% 처치 | 70% 처치 |
|-------|----------|----------|
| 1 (10마리) | 20W + 10S | 14W + 7S |
| 2 (15마리) | 30W + 15S | 21W + 11S |
| 3 (20마리) | 40W + 20S | 28W + 14S |
| 4 (25마리) | 50W + 25S | 35W + 18S |
| 5 (30마리) | 60W + 30S | 42W + 21S |

### 2-C. 생산 건물 수입

**제재소 (Lumber Mill)**:
- DAY 90초 / 5초 간격 = 18 ticks × 2W = **36W per DAY**
- 사이클 N에 지어졌다면 사이클 N부터 매 DAY 36W
- BUILD/NIGHT엔 정지 (DAY only)

**채석장 (Quarry)**:
- DAY 90초 / 8초 간격 = 11 ticks × 2S = **22S per DAY**

> **주의**: 생산 시점은 **DAY 시작부터 카운트** (BUILD 끝나자마자 타이머 0으로 리셋됨)

### 2-D. 누적 수입 시뮬레이션 템플릿

```
[Cycle N] 시작 인벤토리: X W + Y S
  + DAY N 채집: ___W + ___S
  + 제재소 (있으면): +36W
  + 채석장 (있으면): +22S
  ──────────
  BUILD N 직전 인벤토리: ___W + ___S

  - BUILD N 지출: 터렛 N개 × 30W+15S, 벽 M개 × 15W ...
  ──────────
  NIGHT N 직전 인벤토리: ___W + ___S

  + NIGHT N 드롭: 처치율 × wolves_N × (2W+1S)
  ──────────
  [Cycle N+1] 시작 인벤토리: ___W + ___S
```

## ⚔️ 3. 방어 DPS 계산

### 3-A. 단위별 DPS

| 유닛 | 공격력 | 쿨다운 | DPS |
|------|--------|--------|-----|
| 플레이어 | 10 | 1.0s | **10** |
| 기본 터렛 | 8 | 0.8s | **10** |
| 늑대 (몬스터) | 5 | 1.0s | **5** |

### 3-B. 사이클 N 방어 시뮬레이션

**입력**:
- `T` = 보유 터렛 수
- `wolves` = 사이클 N의 늑대 수
- 플레이어 항상 1 (다운 X 가정)

**총 아군 DPS** = `(T + 1) × 10` (터렛 + 플레이어)

**총 적 HP** = `wolves × 20`

**처치 소요 시간** = `총 적 HP / 총 아군 DPS` (단, 터렛 사거리 안에 있어야 함)

**스폰 + 도달 타임라인**:
- 스폰 간격 0.5s. wolves 마리 스폰 완료까지 = `wolves × 0.5s`
- 늑대 이동: 맵 가장자리 → 코어 = ~480px / 64 px/s = **7.5초**
- 첫 늑대 도달 = 7.5s
- 마지막 늑대 도달 = `7.5 + wolves × 0.5` 초

**NIGHT 60초** 안에 모두 처리해야 함.

### 3-C. 사이클별 필요 터렛 추정 (참고)

| Cycle | wolves | 필요 DPS (60s 안 처치) | 필요 터렛 수 |
|-------|--------|----------------------|--------------|
| 1 | 10 (200 HP) | 200/60 ≈ 4 (플레이어로 충분) | 0 (이론) ~ 1 (실전) |
| 2 | 15 (300 HP) | 5 | 1 |
| 3 | 20 (400 HP) | 7 | 1~2 |
| 4 | 25 (500 HP) | 9 | 2 |
| 5 | 30 (600 HP) | 10 | 2 |

> ⚠️ **위 표는 "최소 DPS" 기반**. 현실은:
> - 늑대가 스폰되자마자 바로 사정거리 안 들어오지 않음 (이동 시간 7.5s)
> - 플레이어가 모든 늑대 사거리 안에 있을 수 없음 (분산 스폰)
> - 터렛은 한 번에 1마리만 잡음 → 다구간 분산 처리 필요
> - 그래서 **실전 필요 터렛은 위 표의 2~4배** 잡아야 함

**현실적 추천 사이클별 터렛 수** (게임 디자인 의도):
- Cycle 1: 1~2 터렛
- Cycle 2: 2~3 터렛
- Cycle 3: 3~4 터렛
- Cycle 4: 5~6 터렛
- Cycle 5: 7~9 터렛

→ **이 곡선에 자원 수입을 맞춰야 함**

### 3-D. 코어 HP 소비 계산

- 늑대 1마리가 코어 도달 후 1초당 5 dmg
- 코어 HP 500 → 100 wolf-second면 파괴
- 사이클 5 (30 wolves)가 1초씩 코어 때리면 150 dmg/초 → 3.3초만에 코어 파괴
- 그래서 **사이클 5 늑대가 코어 도달하면 안 됨**

## 🚨 4. 자주 빠뜨리는 계산

1. **시야 확장으로 자원 발견** — 정적 9×9가 아니라 동적 확장. 사이클이 지날수록 발견 가능 자원 늘어남.
2. **mill/quarry 누적 효과** — 사이클 1에 지으면 사이클 5까지 4번 더 작동. 36W × 4 = 144W 추가. 무시 못 할 큰 수입.
3. **NIGHT 드롭 사이클별 증가** — 5 사이클 누적: 10+15+20+25+30 = 100 wolves × 2W = 200W + 100S 총 드롭.
4. **확장 비용 거의 무시 가능** — 처음 8회 = 24W + 8S. 너무 싸서 제약이 안 됨.
5. **돌이 항상 부족** — wood:stone 65:35 비율이라 돌이 더 희소. 터렛 비용에 돌 들어가서 돌이 병목.

## 📈 5. 현재 (2026-04-25) 시뮬레이션 — A+D 적용 후

플레이어가 mill/quarry 우선 + 터렛만 짓는 가정.

```
[Cycle 1]
시작: 30W + 15S
+ DAY 1 채집 (~60W + 30S — 보수)
= 90W + 45S
- BUILD 1: mill (40W+5S) + quarry (15W+20S) = -55W -25S
= 35W + 20S
- BUILD 1 추가: 터렛 1개 (30W+15S) = -30W -15S
= 5W + 5S
+ NIGHT 1 드롭 (10 마리 100% = 20W+10S)
= 25W + 15S

[Cycle 2]
+ DAY 2 채집 (~60W + 30S)
+ mill 36W + quarry 22S
= 25 + 60 + 36 = 121W
  15 + 30 + 22 = 67S
- BUILD 2: 터렛 5개 시도 (5 × 30W+15S = 150W+75S)
  → 못 사. 터렛 4개 (120W+60S) = 1W + 7S
+ NIGHT 2 드롭 (15마리 = 30W+15S)
= 31W + 22S

[Cycle 3]
+ DAY 3 (60+30) + mill 36 + quarry 22
= 31+60+36 = 127W, 22+30+22 = 74S
- BUILD 3: 터렛 4개 (120+60) = 7W + 14S
+ NIGHT 3 (40W+20S)
= 47W + 34S

[Cycle 4]
+ DAY 4 (mid duration 150s — 더 김!)
  채집 ~100W+50S (시간 늘어남) + mill 60W + quarry 37S
= 47+100+60 = 207W, 34+50+37 = 121S
- 터렛 6개 (180W+90S) = 27W+31S, 추가 벽 1개 (15W) = 12W+31S
+ NIGHT 4 (50W+25S)
= 62W+56S

[Cycle 5]
+ DAY 5 (150s) + 생산
= 62+100+60 = 222W, 56+50+37 = 143S
- 터렛 7개 (210+105) = 12W + 38S
- NIGHT 5 (60W+30S)
```

**누적 터렛 수**: 1 → 5 → 9 → 15 → 22

**여전히 너무 풍족**. 추가 컷 필요.
