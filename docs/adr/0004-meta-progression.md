# ADR-0004: 메타 진행 모델 — 롤체식 수평 다양성

**날짜**: 2026-04-27
**상태**: 적용됨

## Context (배경)

Phase 2 마무리·Phase 3 설계 단계에서 "어떤 종류의 다양성·진행감으로 갈지" 결정 필요.

원랜디(랜덤 디펜스) 모델 검토했으나 두 가지 충돌:
- 원랜디 = 골드 랜덤뽑기 + 합성. Survive = 자원 모아 직접 건설(RTS형). 본질적으로 다른 메카닉.
- 원랜디 다양성의 핵심 = 40~50종 SSS 풀. 1인 개발자 첫 게임 스코프를 한참 초과.

캐릭터 성장 BM(가챠·과금 강화) 박는 안도 검토했으나 Core Pillars (특히 ① 수평적 특화)와 충돌.
롤체(TFT) 모델 — "매 판 균등 시작 + 매치 내 운+결정" — 이 디자인 의도와 일치.

## Decision (결정)

### 1. 메타 진행 (판 사이) — X
- 캐릭터·자원·언락 영구 진행 X
- 매 판 완전 리셋
- BM(가챠·과금 강화) 안 박음

### 2. 한 판 내 (매치 내) — O
- 캐릭터 성장 가능
- 4계열 빌드 안에서 캐릭터가 그 계열 능력 흡수
- 자원 분배 trade-off: "타워에 투자 vs 캐릭터에 투자"
- → 같은 메인 빌드여도 매번 다른 변형

### 3. 다양성 메카닉
- 옵션 풀 수(40+종) X — 메인 빌드 5~10개면 충분
- 매 판 다른 메인 강제: **바이옴 자원 분포** (ADR-0005)
- 매 판 보조 변형 강요: **사이클별 적 차별화** (추후 디자인)
- 같은 빌드 내 변형: **자원 분배 trade-off** (위 2번)

## Consequences (영향)

- ✓ BM 안 박고도 매 판 다양성·반복성 확보
- ✓ Core Pillar ① 수평적 특화, ④ 매 런 다양성과 일치
- ✓ 6종 터렛 풀로도 매 판 다른 빌드 가능 (변수 곱셈)
- ✓ 1인 개발자 스코프 적정
- ✗ 메타 진행감(컬렉션·언락) 없음 — 일부 플레이어 타입 어필 약함
- ✗ 매 판 리셋이라 장기 플레이 동기 약함 (Phase 3+ 메타 코스메틱·도전과제로 보완 가능)
- ✗ 4계열 × 캐릭터 변형 디자인 비용 증가
- 🔧 4계열별 캐릭터 능력 모양 (구체 디자인) — OPEN_ISSUES
- 🔧 캐릭터 강화 UI·메카닉 — OPEN_ISSUES
- 🔧 마법 빌드 + 캐릭터 위주 변형 시 마나주입 trade-off 점검 — OPEN_ISSUES
