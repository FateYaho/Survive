# ADR-0005: 바이옴 시스템 — 매 판 자원 분포 절차적 생성

**날짜**: 2026-04-27
**상태**: 적용됨

## Context (배경)

ADR-0004에서 결정한 롤체식 수평 다양성의 핵심 메카닉 = "매 판 다른 메인 빌드 강제".
6종 터렛 풀 한계를 "상황 변수"로 보완해야 매 판 다양성이 살아남.

기존 맵 시스템: 30×30 고정. 바이옴 시스템 도입으로 매 판 자원 분포를 다르게 하면 → 매 판 다른 메인 빌드 강제 가능.

## Decision (결정)

### 1. 자원 분포 — 주력 + 부산물 + 혼합 바이옴
- 각 바이옴 = 주력 자원 1개 + 다른 자원 소량
- 단일 계열 바이옴 4종 (나무·돌·철·마법) + 혼합 바이옴 몇 개 (조합 미정)

### 2. 배치 패턴 — 풍부도 그라데이션 + 위치 랜덤
- 한 바이옴 안에서 가장자리 약함, 중심 풍부
- 바이옴 자체 위치는 매 판 랜덤
- 단 너무 가까이 붙지 않게 적정 거리 보장

### 3. 위험 — 첫 마무리 단순화
- 첫 마무리 버전: 적은 NIGHT 페이즈에만 출현, DAY 탐험은 안전
- Phase 3+: 환경 위험(가스·추위 등) 외곽에 추가하여 자원 vs 위험 trade-off 강화

### 4. 발견 — 시각 차별화
- EXPLORED 시 바닥 색깔로 바이옴 종류 표시
- 자원 노드는 안쪽으로 갈수록 풍부 → 탐험 = 보상

### 5. 절차적 생성 — 시드 기반
- 매 판 다른 시드로 바이옴 배치 생성
- 매 판 다른 맵 = 매 판 다른 메인 빌드 강제

### 6. 테크 결정 시점 — 자유 선택형
- 게임이 정찰 페이즈도, 점진적 결정도 강제 X
- 플레이어 스타일 자유 (도박파/신중파 둘 다 가능)
- 별도 시스템 추가 없음 — 게임 메카닉상 자동으로 두 스타일 다 됨

## Consequences (영향)

- ✓ 매 판 다른 메인 빌드 강제 (ADR-0004 다양성 메카닉의 핵심)
- ✓ Core Pillar ② 능동적 맵 설계, ⑤ 파밍의 탐험적 즐거움 일치
- ✓ 6종 터렛 풀 한계를 상황 변수로 보완
- ✓ 시드 기반이라 "이번 판 망함, 다음 판 다시" 자연스러움
- ✗ 절차적 생성 알고리즘 신규 구현 필요 (바이옴 배치 + 그라데이션 + 거리 보장)
- ✗ 시각 차별화용 바닥 색깔 시스템 신규 (현재 Rectangle 사용 — Phase 2 step 3 시점)
- ✗ 혼합 바이옴 디자인 비용
- 🔧 혼합 바이옴 조합 (어떤 계열끼리 섞는지) — OPEN_ISSUES
- 🔧 바이옴 적정 거리 알고리즘 수치 — 추후 결정
- 🔧 자원 풍부도 그라데이션 수치 — 4계열 보호 메카닉 ADR (추후) 자원 흐름 디자인 시 같이 결정
- 🔧 환경 위험 시스템 — Phase 3+ — OPEN_ISSUES
