# Game Design Document (GDD)
## *(가제) 영토 개척 생존기*

> **한 줄 요약**: 낮엔 영토를 넓히며 자원을 캐고, 밤엔 몰려오는 적으로부터 코어를 지켜라. 수집한 자원으로 특화 테크를 타며 매번 다른 빌드로 생존하는 로그라이크 서바이벌.

**작성자**: Kai
**작성일**: 2026-04-22 (v1.0), 2026-04-24 (v1.1 패치 반영)
**버전**: 1.1 (v1.0 대비 대규모 재설계 — 상세는 `DECISIONS.md` 참조)

### v1.1 주요 변경사항 요약

- 기초 자원 5종 → **4종** (수정 삭제, 토템 전용 재료로 분리)
- 테크 단계 **T0~T4** 체계 확립
- **가공 자원 시스템** (Phase 2+) 신설 — 특화 건물별 생산
- 유닛 소환 시스템 → **삭제** (터렛·시설로 통합)
- 공중 몬스터 → **삭제** (박쥐 제거, 지상 "그림자" 추가)
- 사망 페널티 → **다운 + 30초 부활** (자원 손실 없음)
- 히어로 공격 → **자동 조준** (Vampire Survivors 스타일)
- 맵 크기 **30×30** 통일 (난이도 "큼"=40×40)
- 융합 테크 → **v1.1 제외** (Phase 5+ 재검토)
- 최종 보스 클리어 경로 3개 → **2개** (특화/범용)
- 핵심 수치 → **Phase 1 MVP 실측 후 튜닝** 원칙
- 플랫폼 전략 → **웹 1차 출시** (Phase 3 Vercel/itch.io) 후 모바일·토스인앱 검토

## 1. 게임 개요

### 1.1 장르
- **메인**: 탑다운 서바이벌 + 타워디펜스 + 로그라이크
- **서브**: 리소스 매니지먼트, 빌드 크래프팅

### 1.2 타겟 플레이 시간
- **1 런**: 30~60분 (난이도 설정에 따라)
- **총 플레이**: 메타 성장 + 빌드 다양성으로 20시간+ 리플레이

### 1.3 플랫폼
- **Primary**: 웹 브라우저 (PC 우선)
- **Secondary**: 모바일 이식은 Phase 3 이후 검토

### 1.4 타겟 유저
- Dome Keeper, Slay the Spire, 굶지마 스타일을 좋아하는 플레이어
- 한 판 30~60분에 집중할 수 있는 유저
- 빌드 크래프팅 & 로그라이크 팬

### 1.5 핵심 가치 (Core Pillars)

게임 전반의 의사결정 기준이 되는 5가지 원칙:

1. **수평적 특화**: 테크 트리가 선형이 아닌 계열별 깊이 구조. 플레이어 의도대로 빌드 조립.
2. **능동적 맵 설계**: 토템으로 자원 스폰을 조작해 맵을 "내 전략에 맞게 커스터마이징".
3. **리듬감 있는 긴장**: 낮(계획)과 밤(실행)의 명확한 리듬.
4. **매 런의 다양성**: 맵·자원 분포·이벤트가 매번 달라 같은 빌드도 다르게 플레이.
5. **파밍의 탐험적 즐거움**: 낮 파밍은 단순 노동이 아니라 **탐험 + 수집 + 설계의 통합 경험**. 이를 방해하는 시스템은 도입하지 않음.

## 2. 코어 게임플레이 루프

### 2.1 거시 루프 (한 런 전체)

```
런 시작
  ↓
초기 빌드 페이즈 (캐릭터 선택, 시작 보너스)
  ↓
┌─────────────────────────────┐
│  ☀️ 낮 → 🌙 밤 → 🧘 빌드 페이즈  │  × N사이클
└─────────────────────────────┘
  ↓
최종 보스 (15사이클)
  ↓
런 종료
  ↓
메타 성장 (유물 획득)
```

### 2.2 미시 루프 (한 사이클)

각 페이즈가 명확한 역할을 담당:

| 페이즈 | 역할 | 플레이어 행동 |
|--------|------|--------------|
| ☀️ **낮** | 수집 (Gather) | 채집, 탐색, 영토 확장, 건설 실행, 상점 |
| 🌙 **밤** | 행동 (Act) | 전투, 몬스터 처치로 자원 드랍 |
| 🧘 **빌드 페이즈** | 사고 (Think) | 자원 확인, 테크 연구, 건설 계획, 웨이브 예고 체크 |

### 2.3 런 종료 조건

- **승리**: 15사이클 최종 보스 처치
- **패배**: 코어(Core) HP 0 (유일한 패배 조건)

## 3. 플레이어 캐릭터

### 3.1 조작 방식

- **시점**: 탑다운 2D
- **이동**: WASD 또는 방향키 (8방향)
- **기본 공격**: 좌클릭 유지 시 자동 연속 공격
- **건설**: 우클릭으로 건설 메뉴
- **회피**: 스페이스바 (대시)
- **무기 교체**: 숫자키 1~4 또는 마우스 휠
- **상호작용**: E/F 키

### 3.2 기본 스탯

| 스탯 | 초기값 | 설명 |
|------|--------|------|
| HP | 100 (초안) | 0 시 다운 상태 → 30초 후 자동 부활 |
| 공격력 | 10 (초안) | 무기별 배율 적용 |
| 공격 속도 | 1.0회/초 | 초당 공격 횟수 |
| 이동 속도 | 4타일/초 | 기준값 |
| 채집 속도 | 1.0 | 기준값 |
| 시야 | 5타일 반경 | 어둠에서 보이는 거리 |

> ⚠️ 모든 수치 초안. Phase 1 MVP 실측 후 튜닝 (§15.1). 상세 TBD 목록은 `OPEN_ISSUES.md` 참조.

**공격 조준 방식**: **자동 조준** (Vampire Survivors 스타일). 매 공격마다 가장 가까운 몬스터 자동 선택. 플레이어는 이동·회피에 집중.

### 3.3 사망 페널티 (v1.1 전면 교체)

- HP 0 → 코어 위치에 **다운 상태**로 누움 (컨트롤 불가, 무적)
- **30초 후 자동 부활** (HP 100% 복구)
- **자원 손실 없음**
- 다운 중에도 방어 시설(터렛·벽)은 정상 작동 → **시설 기반 방어선이 코어를 지킨다**
- 다운 중 코어 HP 0 → 게임 오버
- **코어 파괴가 유일한 게임 오버 조건** (플레이어 사망 ≠ 게임 오버)

> ⚠️ 30초는 초안. Phase 1 플레이 테스트로 튜닝. 사이클별 차등(초반 15초 / 후반 45초) 검토 중.

### 3.4 캐릭터 종류 (5종)

각 캐릭터는 특정 계열의 "얼굴" 역할:

| 캐릭터 | 해금 조건 | 특징 | 추천 빌드 |
|--------|---------|------|----------|
| ⛏️ **광부** | 기본 | 채집 속도↑, 전투 평이, 더블 픽 | 전 계열 가능, 특히 마법 |
| ⚔️ **전사** | 5런 클리어 | 체력↑↑, 공격력↑, 광역 휩쓸기 | 돌 (골렘) |
| 🛠️ **엔지니어** | 특정 업적 | 건설 비용 -20%, 자동 수리 드론 | 철 (산업) |
| 🧙 **마법사** | 금 계열 최초 해금 | 마법 공격력↑↑, 원소 연쇄 | 금/수정 (마법) |
| 🌿 **드루이드** | 히든 조건 (TBD) | 밸런스, 주변 나무 계열 터렛 효과 +20% | 나무 (자연) |

### 3.5 테크별 컨트롤 편향

각 빌드마다 플레이어의 역할 비중이 다름 (**편향 설계** — 완전 분리 X):

| 테크 | 플레이어 역할 | 시설 역할 | 플레이 느낌 |
|------|-------------|----------|-----------|
| 🪵 나무 | 딜러 (강) | 탱커 보조 | 내가 쏘고 **정령 터렛이** 막아줌 |
| 🪨 돌 | 탱커 | 딜러 (강) | 내가 시선 끌고 **룬 포병·요새가** 부숨 |
| ⚙️ 철 | 균형 | 자동화 딜러 | 내가 전선 지키고 **레일건이** 자동 딜 |
| ✨ 마법 | 슈퍼딜러 | 최소 방어 | 내가 광역기로 쓸어버림 |

### 3.6 무기 시스템

**무기 슬롯**: 3~4개
- 근접 무기 (기본)
- 원거리 무기 1 (활/총)
- 원거리 무기 2 (테크 특수)
- 회피/이동기

**테크별 주무기**:
- 🪵 정령의 활 (관통형, 리젠)
- 🪨 룬 해머 (근접 광역, 스턴)
- ⚙️ 레일건 (고 DPS, 일직선)
- ✨ 파이어볼/아이스볼 (원소, 광역)

## 4. 맵 & 영토 시스템

### 4.1 맵 구조

- **기본 30×30** (모든 Phase 공통 기본값)
- 난이도 슬라이더 "큼" 선택 시 40×40 (Phase 3+)
- Phase 1 MVP: 30×30 고정
- 중앙에 **코어** 배치
- 초기 공개: 코어 주변 5×5
- 나머지는 **안개(Fog)** 로 덮임

### 4.2 타일 상태

| 상태 | 설명 | 플레이어 권한 |
|------|------|--------------|
| 🌫️ 미탐색 | 안개, 내용 숨김 | 접근 불가 |
| 👁️ 탐색됨 | 자원·몬스터 표시, 내 땅 아님 | 탐사 가능, 건설 불가 |
| 🏠 내 영토 | 해금 완료 | 채집·건설 가능, **방어 영역** |

### 4.3 바이오옴

| 바이오옴 | 주 자원 | 적 특성 |
|---------|--------|--------|
| 🌳 숲 | 나무 풍부 | 늑대, 약한 잡몹 |
| ⛰️ 산악 | 돌·철 풍부 | 골렘, 멧돼지 |
| 🏜️ 사막 | 금·수정 풍부 | 중~강 몬스터 |
| 💀 폐허 | 희귀 자원·유물 | 최상위 몬스터 |

### 4.4 영토 = 방어 영역 통합 규칙

**핵심 원칙**:
1. 플레이어가 확장한 영토 안은 **자동으로 방어 대상**
2. 몬스터는 **영토 경계에서 침입 시도**
3. 영토 경계 = 방어선 (벽·터렛으로 강화 필요)
4. 영토 확장 = 공격적 의미 + 방어적 의미 **이중 결정**

이 규칙으로 "바이옴 근처 건물 배치" vs "방어"의 딜레마를 해결.

### 4.5 확장 비용 (거리 기반)

> ⚠️ **이 거리 비용표는 Phase 2부터 적용.** Phase 1 MVP에서는 **코어 영토에 인접한 타일만 1칸씩 확장 가능** (중앙 5×5 → 6×6 → 7×7 식). Phase 2부터 인접하지 않은 원거리 타일도 즉시 해금 가능해지며, 이때 거리 기반 비용이 적용됨.

| 거리 (코어 기준) | 비용 |
|----------------|------|
| 1칸 (인접) | 나무 10, 돌 5 |
| 2칸 | 나무 30, 돌 20, 철 5 |
| 3칸 | 나무 80, 돌 60, 철 20, 금 5 |
| 4칸 | 나무 200, 돌 150, 철 60, 금 20 |
| 5칸+ | 기하급수 증가 |

> 수치는 초안. Phase 2 진입 시 재검토.

### 4.6 플레이 스타일 다양성

영토 구조에 따른 3가지 전략:

**집중형 (초보자)**
- 코어 주변에 모든 건물
- 방어 쉬움, 시너지 낮음

**뻗기형 (중급자)**
- 코어 → 회랑 → 바이옴 허브
- 바이옴 근처 건물 = 채집 효율↑

**허브형 (숙련자)**
- 다방향 허브 확장
- 모든 시너지 극대화, 방어 극난이도

## 5. 자원 시스템

### 5.1 기초 자원 (4종)

| 자원 | 주 획득처 | 역할 |
|------|----------|------|
| 🪵 나무 | 숲 | 나무 계열 테크 |
| 🪨 돌 | 산악 | 돌 계열 테크 |
| ⚙️ 철 | 산악·철광 | 철 계열 테크 |
| ✨ 금 | 사막·폐허 | 마법(금) 계열 테크 |

> 💠 **수정(Crystal)**은 이 표의 기초 자원에서 제외됨. 수정은 별도로 채집 가능하지만 4계열 테크 연구엔 사용되지 않고, **토템 제작 전용 재료**로 쓰임 (§7 참조).

### 5.2 채집 속도 (기준: 맨손 도구)

| 자원 | 1회 채집 시간 | 획득량 | 1분당 획득 |
|------|--------------|--------|-----------|
| 나무 | 1초 | 2개 | 120개 |
| 돌 | 1.5초 | 2개 | 80개 |
| 철 | 2초 | 1개 | 30개 |
| 금 | 3초 | 1개 | 20개 |

> 수정은 토템 전용 재료로 분리됨 (§7). 채집 시간은 초안, Phase 1 튜닝 대상.

### 5.3 도구 업그레이드

- T1 도구: 기준 그대로
- T2 도구: 속도 +30%, 획득량 +50%
- T3 도구: 속도 +60%, 획득량 +100%

### 5.4 가공 자원 (Phase 2+)

각 계열의 특화 건물은 건설된 뒤 **매 밤 종료 시 가공 자원 1개 생산**. T3~T4 테크 연구의 필수 재료.

| 생산 건물 | 가공 자원 | 용도 |
|----------|----------|------|
| 🌳 정령의 숲 | **정령 수액** | 나무 T3~T4 연구 |
| ⚒️ 대장간 | **룬 파편** | 돌 T3~T4 연구 |
| 🏭 공장 | **기계 부품** | 철 T3~T4 연구 |
| 🔮 마법 연구소 | **마나 결정** | 마법 T3~T4 연구 |

**규칙**:
- 밤 동안 건물 파괴 시 해당 밤 생산분 없음 → **건물 방어 동기 강화**
- 생산량은 건물당 사이클당 1개 (초안). Phase 2 플레이 테스트로 튜닝.
- ⚠️ Phase 1 MVP에서는 가공 자원 시스템 **미구현**. 정령의 숲은 시설 효과(+20% 채집) + T2 해금 gate만.

> "고목", "생명의 나무", "세계수" 같은 이름은 **테크 단계의 명칭**이지 자원이 아님. 해당 테크가 해금되면 플레이어/터렛 효과가 바뀌는 것이지, "고목"이라는 아이템이 인벤토리에 쌓이지 않음. 자세한 테크 단계는 §6.3 참조.

## 6. 테크 & 건물 시스템

### 6.1 핵심 원칙 (v1.1 재설계)

- **T0~T4 5단계 체계** (T0=원재료, T4=정점)
- **원재료만으로 T1~T2 연구** / **원재료 + 가공 자원 조합으로 T3~T4 연구**
- **3범주 연구 옵션** (무기 / 스탯·장비 / 방어 시설) 중 자원 부족으로 부분 선택 → **빌드 다양성의 근원**
- 건물 자체가 시설 효과 + T3~T4 전용 가공 자원 생산
- 공간·자원 제약이 전략성 확보 → 턴제 불필요

### 6.2 테크 진행 흐름

```
T0 (원재료)
  └─ 자원 타일에서 채집 (나무/돌/철/금)

T1 (기본 가공)
  └─ 연구실에서 원재료만 소비 → 즉시 적용
  예: 단단한 나무, 돌도끼, 나무 벽, 기본 터렛

T2 (특화 진입)
  └─ 해당 계열 특화 건물 건설 (정령의 숲/대장간/공장/마법 연구소)
  └─ 건물 자체가 시설 효과 + 연구 gate

T3~T4 (중급·정점)
  └─ 특화 건물에서 연구: 원재료 + 해당 건물이 밤마다 생산한 가공 자원
  예: 나무 T3(생명의 나무) = 나무 300 + 정령 수액 3
```

> ⚠️ **융합 테크는 v1.1에서 전면 제외.** 각 계열 T0~T4 독립 트리 깊이 파는 구조에 집중. Phase 5+ 확장 재검토 (`OPEN_ISSUES.md` P3).

#### 3범주 연구 옵션

각 티어에서 복수 연구 옵션 제공. 한 런에 자원 부족으로 전부 불가능 → **빌드 다양성**.

| 범주 | 내용 |
|------|------|
| 1. 플레이어 무기 | 주력 무기 업글 / 신무기 해금 |
| 2. 플레이어 스탯·장비 | HP·공격력·리젠·방어력 영구 보너스 |
| 3. 방어 시설 | 터렛 / 벽 / 광역 시설 (T3+는 **특수 스킬** 부여) |

### 6.3 4계열 테크 트리

#### 🪵 나무 계열 — 자연/생명
```
T0 나무 → T1 단단한 나무 → T2 고목 → T3 생명의 나무 → T4 세계수
                                ↓
                        정령 터렛 (자동 공격 + 아군 회복)
```
- **특징**: 자가 회복 시설, 터렛 방어, 리젠 버프
- **약점**: 낮은 DPS, 중장갑 적에게 취약

#### 🪨 돌 계열 — 골렘/대지
```
T0 돌 → T1 단단한 돌 → T2 마력석 → T3 룬스톤 → T4 고대 성체
                            ↓
                    룬스톤 요새 (광역 실드)
```
- **특징**: 최고 방어력, 광역 스턴
- **약점**: 느림, 속도형 적에게 약함

#### ⚙️ 철 계열 — 기계/산업
```
T0 철 → T1 강철 → T2 합금 → T3 미스릴 → T4 아다만티움
                      ↓
                자동 공장 (자원 자동 생산)
```
- **특징**: 고 DPS, 자동화
- **약점**: 마법 계열 적에게 취약, 초기 비용 큼

#### ✨ 금 계열 — 마법/아케인
```
T0 금 → T1 정제된 금 → T2 마력 결정 → T3 신성 결정 → T4 창조의 보석
                                            ↓
                                    차원 결계 (무적 시간)
```
- **특징**: 원소 공격, 상태이상, 특수 스킬
- **약점**: 지속력 없음, 자원 소모 극심

> 📝 v1.0에 있던 "금/수정 계열"은 **"금 계열"** 로 정리 (수정은 토템 전용 재료로 분리).

#### T3/T4 터렛 특수 스킬 매트릭스

**T3/T4 터렛은 고유 특수 스킬 보유**. 방어 시설 범주의 질적 도약.

| 계열 | T3 터렛 | T4 터렛/시설 |
|------|---------|-------------|
| 🪵 나무 | **정령 터렛** — 공격 + 주변 건물 HP 회복 | **세계수 코어** — 맵 전역 자연 리젠 버프 |
| 🪨 돌 | **룬 포병** — 광역 공격 + 스턴 | **고대 성체** — 광역 실드 (몬스터 일시 차단) |
| ⚙️ 철 | **레일건** — 고DPS 일직선 관통 | **자동 공장 터렛** — 방어 + 주변 자원 자동 생산 |
| ✨ 마법(금) | **원소 포탑** — 지속 화염/빙결 지대 | **차원 결계** — 특정 영역 일시 무적 |

### 6.4 건물 예시 (MVP 범위)

| 건물 | 비용 | HP | 시설 효과 | 해금되는 테크 |
|------|------|-----|----------|--------------|
| 🏛️ 연구실 | 나무 50, 돌 30 | 200 | T1 테크 해금 | 기본 도구·벽·터렛 |
| 🌳 정령의 숲 | 나무 100, 돌 50 | 150 | 주변 나무 채집 +20% + 정령 수액 생산 | 나무 T2~T4 |
| ⚒️ 대장간 | 돌 100, 철 20 | 300 | 주변 돌·철 채집 +20% + 룬 파편 생산 | 돌 T2~T4 |
| 🏭 공장 | 철 80, 돌 80 | 250 | 자동 채집기 가동 + 기계 부품 생산 | 철 T2~T4 |
| 🔮 마법 연구소 | 금 40 | 150 | 주변 금 채집 +20% + 마나 결정 생산 | 마법 T2~T4 |

### 6.5 바이오옴 보너스 (Phase 2+)

→ "위험 감수하고 바이오옴 근처 가는 플레이"에 보상

## 7. 토템 시스템 (Phase 2)

### 7.1 기본 규칙

- 내 영토 타일에 설치, 주변 N타일에 영향
- **안개 걷힌 타일의 새 자원 생성** 시 확률 가중치 조작
- **토템 제작은 전적으로 "수정(Crystal)"을 소비.** 수정은 §5에서 기초 자원에서 제외됐지만, **사막·폐허 바이오옴에서 별도 채집** 가능. 토템 이외 용도 없음.

### 7.2 토템 티어

| 티어 | 비용 예시 | 효과 |
|------|----------|------|
| 🌱 T1 | 수정 5 + 해당 계열 원재료 소량 | 주변 타일 해당 자원 +40% |
| 🌲 T2 | 수정 15 + 원재료 + 가공 자원 1 | 상위 자원 스폰 +10% 추가 |
| 🌟 T3 | 수정 40 + 원재료 + 가공 자원 3 | 반경 7타일 축복, 효과 +50% |

> 수치는 초안. Phase 2 플레이 테스트로 튜닝.

### 7.3 핵심 규칙

- **중첩 감소**: 같은 토템 여러 개 시 체감 감소 (40% → 25% → 15%)
- **상충**: 다른 계열 토템 겹치면 서로 상쇄 → 구역 분리 전략
- **제거 비용**: 해체 시 50%만 환불
- **이벤트 연동**: 유성우 시 수정 토템 2배 등

### 7.4 건물과의 시너지

나무 토템 + 정령의 숲 = 나무 특화 구역 형성 → 플레이어 고유 "왕국" 느낌

☀️ 낮 (고정 시간)
  ↓

🧘 빌드 페이즈 (무제한)

### 8.2 낮 페이즈

**기본 시간**:
- 1~3사이클: 1.5분
- 4~9사이클: 2.5분
- 10~15사이클: 3분

**할 수 있는 것**:
- 자원 채집
- 영토 확장
- 건설 실행 (빌드 페이즈에서 계획한 것)
- 상점 이용 (상인 NPC)
- 맵 탐색

**자동 채집기**: T2 철 계열 해금 시 → 중반부터 파밍 효율 상승

### 8.3 밤 페이즈

**기본 시간**:
- 1~3사이클: 1분
- 4~9사이클: 1분
- 10~15사이클: 1분

**할 수 있는 것**:
- 몬스터 처치 (드랍 획득)
- 코어·건물 방어
- 터렛 수리
- 플레이어 직접 전투

**시간 종료 시**: 남은 적 사라짐 (평화 복귀)

### 8.4 빌드 페이즈 (심플 버전 - Option A)

**진입**: 밤 종료 직후 자동
**종료**: 플레이어의 "준비 완료" 버튼
**시간 제한**: 없음

**UI 구성**:
```
┌─────────────────────────────────────┐
│  🌅 다음 날을 준비하세요               │
│                                      │
│  [내 자원]                           │

- 다음 웨이브 예고 열람

**왜 밤 후에?**: 긴장(밤) → 이완(빌드) → 이완(낮) → 긴장(밤) 리듬 유지

#### 자발적 종료 유도 연출 (Phase 3~4 도입)

빌드 페이즈는 시간 제한 없음 (§8.1 유지). 단 "긴장→이완 리듬"(Core Pillar 3) 유지를 위해 **플레이어가 자발적으로 빨리 끝내도록 유도**하는 연출 장치:

- **BGM 볼륨 점진 감소**: 진입 시 잔잔 → 2분 후 볼륨 절반 → 3분 후 완전 무음
- **"준비 완료" 버튼 강조 애니메이션**: 1분 뒤부터 미묘한 반짝임, 2분 뒤 화면 중앙으로 슬라이드
- **후반 사이클(10~15) 우상단 경고**: "다가오는 위협" 시각 연출(구름 짙어짐 등)
- **3분 경과 리마인더**: 코어에서 "수호자여, 어둠이 다가옵니다" 소프트 보이스 (스킵 가능)

> ⚠️ 구체안은 `OPEN_ISSUES.md` P1 검토 중.

## 9. 웨이브 & 예고 시스템

### 9.1 3층 웨이브 구조

#### 🟢 Layer 1: 평시 웨이브 (매 밤)
- 혼합 편성, 여러 타입 섞임
- 대부분 빌드 대응 가능

- 특정 타입 집중 편성
- 빌드 상성 발생
- 예: 5사이클 기계 군단, 10사이클 어둠 무리

#### 🔴 Layer 3: 최종 보스 (15사이클)
- 모든 타입 혼합 + 특수 패턴
- 4부위 약점 시스템 (섹션 12 참조)

### 9.2 3단계 예고 시스템

```
[3 사이클 전] 📜 예언: "5사이클 후, 기계 군단 옵니다"
   → 테크 루트 변경 가능 (전면 빌드 수정)

[1 사이클 전] ⚠️ 경고: "다음 밤, 기계 군단!"
   → 터렛 추가, 무기 교체, 아이템 구매 시간

[밤 시작 10초 전] 🔔 카운트다운
   → 최종 배치 점검
```

### 9.3 예고 시스템의 전략적 의미

### 10.2 이동·공격 패턴 (v1.1 단순화)

**이동 패턴 2종**:

**타이밍**:

### 10.4 타깃팅 규칙 (v1.1 3주체 명시)

#### 1. 몬스터 → 공격 대상 결정

- 영토 경계 진입 시점에 **"가장 가까운 공격 대상"을 타깃으로 고정**
- 대상 종류 무관 (플레이어·건물·코어 중 거리만 판단)
- **타깃 사망 또는 공격 사거리 이탈** 시에만 재탐색
- 매 프레임 재탐색 X → 구현 단순 + 플레이어의 "미끼 유도" 전략 가능

#### 2. 히어로 → 몬스터 (자동 공격)

- **자동 조준** (Vampire Survivors 스타일)
- 매 공격마다 **가장 가까운 몬스터** 재탐색
- 수동 조준 불필요 → 플레이어는 이동·회피에만 집중

#### 3. 방어 터렛 → 몬스터

- 사거리에 처음 진입한 몬스터 **타깃 고정**
- 타깃 사망 또는 사거리 이탈 시 재탐색
- **몬스터와 동일한 규칙**: 구현 일관성

## 11. 처치 보상 & 상점

### 11.1 처치 보상 (밤 시간제 + 드랍)

밤 웨이브는 **시간제**이지만, 몬스터 처치 시 **자원 드랍**. 시간 내 얼마나 사냥할지 플레이어 선택.

### 11.2 드랍 테이블 (원칙)

**기본 원칙**: 그 몬스터 테마의 자원을 줌

| 몬스터 | 드랍 | 의미 |
|--------|------|------|
| 🐺 늑대 | 가죽, 소량 나무 | 일반 |
| 🐗 멧돼지 | 가죽, 소량 돌 | 일반 |
| 👤 그림자 | 소량 수정 | 희귀 약간 (토템 재료) |
| 👻 유령 | 영혼 파편 | 마법 재료 |
| 🤖 기계 몬스터 | **철 大**, 금 소량 | 주자원 + 평소 못 얻는 자원 소량 |
| 🌑 어둠 몬스터 | **어둠 결정**, 나무·수정 소량 | 특수 + 맛보기 |
| 🌋 원소 몬스터 | **원소 정수**, 돌·철 소량 | 특수 + 기본 보강 |

### 11.3 핵심 설계: 특화 빌드에도 "평소 못 얻는 자원 소량"

나무 특화 빌드도 테마 웨이브에서 **금·수정 소량** 획득 가능:
→ 하이브리드 빌드(2계열 동시 활용) 진입로 확보
→ 수정 드랍은 토템 시스템으로 이어짐 (§7)

> ⚠️ v1.0에 있던 "융합 테크 옵션 확보" 근거는 v1.1에서 융합 테크 제외로 삭제. Phase 5+ 재도입 시 복구.

### 11.4 드랍량 원칙

```
밤 한 번의 총 드랍량 = 해당 사이클 낮 파밍량의 30~50%
```

→ 낮 파밍이 메인, 밤 사냥이 보너스
→ 둘 다 해야 테크 완성 가능한 자원량

### 11.5 1회용 아이템 상점

낮 페이즈 동안 상인 NPC 등장. 재고 제한.

**📊 방어형 (범용)**

| 아이템 | 효과 | 비용 |
|--------|------|------|
| 긴급 방벽 | 코어 주변 임시 벽 3개 | 나무 50 |
| 원소 저항 포션 | 다음 웨이브 원소 저항 +50% | 금 20 |
| 수리 드론 | 밤 동안 시설 자동 수리 | 철 30, 수정 5 |

**⚔️ 공격형 (빌드별)**

| 아이템 | 효과 | 비용 |
|--------|------|------|
| 🪵 정령의 축복 | 1웨이브 공격력·속도 +30% | 나무 100, 금 10 |
| 🪨 룬 지진 | 웨이브 시작 시 광역+스턴 | 돌 150, 수정 10 |
| ⚙️ 기계 증원 | 자동 터렛 3개 1회용 | 철 100, 금 20 |
| ✨ 파멸의 유성 | 원하는 위치 광역 | 금 30, 수정 20 |

### 11.6 플레이어 선택 예시

테마 웨이브 대응 방식 선택:
- **빌드 수정**: 3사이클 전 예언 보고 테크 전환

- **방어 강화**: 벽·터렛 보강

```

**위협 스케일 비율**
```
N사이클 몬스터 총 위협 = 플레이어 최대 출력 × 0.8
```

> ⚠️ 위 3가지 비율은 설계 의도값. Phase 1 MVP 실측 데이터로 검증 후 재조정.

### 15.5 수치 관리 원칙

```typescript
// 설정 상수로 관리
const DAY_DURATION = {
  early: 90,    // 1.5분
  mid: 150,     // 2.5분
  late: 180     // 3분
};
```

플레이 테스트로 즉시 조정 가능하게.

### 15.6 시설 DPS 기여도 (v1.1 신규 항목)

**밸런싱 핵심 지표** — 플레이어 단독 DPS 대비 시설이 밤 방어에 얼마나 기여하는지.

| 측정 항목 | 기준 |
|----------|------|
| 플레이어 단독 DPS × 밤 시간 | vs 해당 사이클 몬스터 총 HP |
| 시설 DPS × 밤 시간 | vs 위와 동일 |
| **플레이어 : 시설 DPS 비율** | **Phase 1 측정 후 결정 (TBD)** |

Phase 1 MVP에서는 시설 = 기본 터렛 1종만 있어서 최소 비율 측정 가능. 사이클별 목표 비율(초반/후반 차등)은 **실측 데이터 확보 후 수립**.

## 16. 밸런스 원칙

### 16.1 학살 빌드 자기 조절

**원칙**: 인위적 페널티 X, **보스전 구조로 자동 조절**

- 광역기 = 일반몹 유리, 보스 단일 DPS 약함
- 자원 많이 얻지만 → 보스 약점 공격 어려움
- → 자원으로 단일 DPS 테크 전환 필요 → 자연 밸런스

| 구분 | 단일 HP | 수 | 광역 효율 | 단일 효율 |
|------|---------|-----|----------|----------|
| 일반 몹 | 낮음 | 많음 | 10배 | 1배 |
| 테마 몹 | 중간 | 보통 | 5배 | 1배 |
| 최종 보스 | 매우 높음 | 1 | 0.2배 | 5배 |

### 16.2 파밍/사냥 강제

**원칙**: 테크 완성 자원 = 낮 파밍 + 밤 사냥 (둘 다 필수)

- 낮만: 부족 → 빌드 미완성
- 밤만: 더 부족
- 둘 다: 클리어 가능

선택 가능한 비율:
- 파밍 중심: 90:10 (안전)
- 밸런스: 70:30 (기본)
- 사냥 중심: 50:50 (공격적)

### 16.3 테마 웨이브 자원 설계

**원칙**: 주 드랍(大) + 평소 못 얻는 자원(소량)

- 주 드랍 → 해당 테마 직결 자원
- 보조 드랍 → **빌드 다양화 기회** (하이브리드 2계열 진입)
- 극한 특화 플레이어도 보조 계열 맛보기 가능

> 📝 v1.0의 "융합 옵션 확보"는 융합 테크 v1.1 제외로 축소. Phase 5+ 복귀 시 재반영.

### 16.4 빌드 다양성 축 (7개)

1. 테크 특화 선택 (4계열 × 편향 설계)
2. 확장 방향 선택 (바이옴)
3. 토템 배치 (Phase 2)
4. 1회용 아이템 운용
5. 밤 행동 선택 (방어 vs 사냥)
6. 캐릭터 선택 (5종)
7. 영토 구조 (집중/뻗기/허브)

### 16.5 설계 가이드라인

> **"파밍의 탐험적 즐거움을 해치는 시스템은 도입하지 않는다"**

- 수련 시스템 등 "효율적 최적화"가 파밍을 밀어내는 구조 배제
- 플레이어가 "다음 타일에 뭐가 나올까?" 설레도록 유지

## 17. 세계관 & 아트

### 17.1 세계관: 고대 판타지

> *"대지의 코어가 깨어나 세상을 파괴할 때, 당신은 마지막 수호자로 선택됐다. 고대의 숲과 산에서 자원을 모아 코어를 다시 봉인할 때까지 15일의 시간만이 남았다."*

**설정**:
- 중세 판타지 + 신비
- 숲: 정령 / 산: 골렘 / 사막: 잊혀진 문명
- 몬스터 = "어둠의 침식" (코어가 약해지며 풀려난 존재)
- 플레이어 = 수호자

**테크 매핑**:
- 🪵 나무 → 정령·드루이드 마법
- 🪨 돌 → 룬·골렘 마법
- ⚙️ 철 → 드워프·대장장이 기술
- ✨ 마법 → 아케인·고대 마법

### 17.2 아트 스타일: 픽셀 아트 + 무료 에셋

**방식**: 점진적 커스텀 전환

**MVP (Phase 1~2)**
- Kenney.nl 또는 itch.io 무료 팩
- 게임 재미 검증 우선
- "임시 아트" 명시 후 배포

**정식 (Phase 3~4)**
- 시그니처 요소만 커스텀 (캐릭터·코어·토템)
- 배경 타일은 무료 에셋 유지

### 17.3 아트 방향성

**픽셀 크기**: 16×16 권장

**색상 팔레트**
- 낮: 따뜻한 톤 (숲 녹색, 돌 회색, 하늘 파랑)
- 밤: 차가운 톤 (어둠 보라, 달빛 시안, 위협 빨강)
- 테크별: 🪵 녹색 / 🪨 갈색 / ⚙️ 주황 / ✨ 보라

**참고 게임 비주얼**
- Stardew Valley (따뜻한 톤)
- Terraria (모험감)
- Hades (판타지 + 어둠)

### 21.3 시장 포지셔닝

**한 줄 포지셔닝**
> "Dome Keeper의 듀얼 루프 + Slay the Spire의 빌드 다양성"

Dome Keeper가 "간결한 듀얼 루프"로 성공했지만 "얕은 콘텐츠·반복성"으로 비판받은 지점을, 본 게임은 **수평 특화 테크 + 토템 기반 맵 커스터마이징 + 건물 기반 영토 설계**로 해결한 차기 버전을 지향.

### 21.4 시장 기회 & 위협

**기회**
1. Dome Keeper 성공 = 장르 수요 검증
2. 한국어 리뷰 "매우 긍정적" = 한국 시장 존재
3. 2인 팀 성공 = 개인 프로젝트 시장성 있음
4. Dome Keeper 비판 지점 = 본 게임 차별화 기회

**위협**
1. Dome Keeper 후속 업데이트
2. 장르 신작 진입
3. 차별화 실제 작동 리스크 → MVP 검증 필수
