- [ ] **4계열 보호 메카닉 ADR 작성** — 바이옴 자원 흐름 정리 후 (계획: ADR-0006)
  - 결정 사항 메모 (2026-04-27 세션, ADR-0006 박을 때 정식 옮김):
    - 나무: 시간 자동회복 (24시간, 전투 중에도 조금씩) — 회복량 적음
    - 돌: 자원 즉시 큰 회복 (능동 클릭) — 밤엔 수리 X
    - 철: 드론 자동수리 (논리적 SCV — 물리 이동 X, 우선순위 큐 + 수리 중 타워 위 드론 visual) + 옵션 (b) 수리 시 자원 추가 소모 — 밤엔 수리 X
    - 마법: 모드 토글 (마나주입 모드 ↔ 전투 모드) — 마나주입 모드는 캐릭터 한 자리 고정·적 공격 X
    - 4사분면 비용 매트릭스: 시간(나무) / 자원(돌) / 캐릭터(마법) / 인프라(철)
    - 돌·마법 밤 작동: "가능하지만 자연스러운 trade-off로 어려움" (전투 중 동시 수리 어려움)
    - 캐릭터 성장 결합: 4계열 빌드 안에서 캐릭터에 자원 투자 시 그 계열 능력 흡수 (자원 분배 trade-off, ADR-0004 참조)
  - 추가 결정 사항 (2026-04-27 세션 후반, 자원 구조 + 캐릭터 성장 구체화):
    - **자원 구조**: 1티어 2종(나무·돌) + 2티어 4종(강화 나무·강화 돌·철·금) = 총 6종. 비대칭 (나무·돌만 1티어 있음, 철·금은 2티어부터 시작).
    - **1티어 (나무·돌)**: 능력 X. 용도 = 기본 벽·타워 + 영웅 업글 메인 + (검토중) 아이템 메인 재료
    - **2티어 (강화 나무·돌·철·금)**: 특수 효과 보유. 용도 = 4계열 메인 빌드 (특수 효과 타워) + 영웅 4계열 색깔 흡수 (소량) + 아이템 부재료 (소량)
    - **결의 차이**: 1티어 = 양 위주 (싸고 약한 타워 다수) / 2티어 = 질 위주 (비싸고 강한 타워 소수)
    - **FORGE/FACTORY 처리**: (C) 그대로, 의미 약화 — 자동 생산은 백업용 소량 / 바이옴 채집이 메인
    - **캐릭터 성장 (iii) 모델**: 1티어 자원으로 기본 영웅 성장 (메인) + 2티어 자원 (소량) 투자 시 4계열 색깔 흡수
    - **영웅 장착 아이템**: 별개 시스템 X. 능력치 업그레이드로 통합 (본질 같음)
    - **랜덤 타워 뽑기 폐기**: ADR-0004 "직접 건설" 모델과 부분 도입 시 모순 (1티어 능력 없어 의미 약함, 2티어 업그레이드까지 가면 진짜 원랜디 됨)
    - **빌드 결 두 갈래**: (1) 1티어 영웅 액션 빌드 — 1티어 자원 위주, 영웅 + 기본 타워, 영웅 컨트롤로 깨는 결. (2) 4계열 메인 빌드 — 2티어 자원 위주, 특수 효과 타워, 영웅 보조.
